# Scorched Earth Web — Phase Roadmap

**Status:** Draft, awaiting user approval.
**Date:** 2026-05-22.
**Companion docs:** `SPEC.md` (long-form north star), `2026-05-22-phase-1-multiplayer-skeleton-design.md` (first phase detailed).

## Vision (one line)

A faithful, multiplayer browser reimplementation of Wendell Hicken's 1991 *Scorched Earth* with modern, "fun" graphics — built phase-by-phase so every phase ends in a more playable game than the last.

## Cross-phase invariants

These rules apply to every phase. Every spec should reference and respect them.

### 1. Determinism

- All RNG goes through a **seeded PRNG** in `packages/game`. No `Math.random()` anywhere in game logic.
- All randomness sources (terrain gen, wind, AI choices, turn order) derive from `match.seed`.
- Match seed is recorded; replays in Phase 11 require exact reproduction.

### 2. Authoritative server

- The server in `apps/server` owns the canonical state.
- Clients send **intents** (`AIM`, `FIRE`, `BUY`, etc.); they never compute damage or terrain changes themselves.
- Client-side prediction is visual only — never modifies replicated state.

### 3. Pure `packages/game`

- No imports of `pixi.js`, `colyseus`, Node built-ins, or environment globals.
- All functions are pure (input → output) or operate on explicit mutable inputs (no global state).
- Same input must produce identical output across server and client (modulo IEEE-754 quirks; document any platform-dependent behavior).

### 4. State vs. events split

- **Schema state** (replicated automatically by Colyseus) holds the canonical *current* state.
- **Broadcast events** (one-shot messages) carry transient data — trajectory samples, animation triggers, chat lines.
- Rule of thumb: if a late-joiner needs it to reconstruct the match, it's state. If it's a "this just happened," it's an event.

### 5. Append-only terrain mutation log

- Terrain is generated client-side from `terrainSeed`.
- Server broadcasts a list of `CarveOp`s as state; clients replay them locally.
- This stays cheap: a 1600-column heightmap as raw data would be ~3 KB per state delta; carve ops are <50 bytes each.

### 6. Phase additivity

- Each phase ADDS scope, never replaces it. Once Phase N's features ship, Phase N+1 must not break them.
- Each phase's spec defines new schema fields and migrations explicitly. Old saved matches (post Phase 11) must be replayable across versions or fail loudly.

### 7. Test discipline

- All new logic in `packages/game` is TDD-first (test before implementation).
- Coverage targets: `packages/game` ≥90%, `apps/server` ≥70%, `apps/client` smoke-only.
- Every phase ends with a passing CI run.

### 8. Visual fidelity ratchet

- Phase 1 ships "Cartoon Illustrative" at placeholder fidelity.
- Phases 2–7 stay at placeholder fidelity (focus on mechanics).
- Phase 8 is the polish phase — all assets get their final art treatment.
- No phase ships *uglier* than the prior phase (no regressions).

## The phases

Each phase is its own brainstorm → spec → plan → implementation → review cycle.

| # | Phase | Scope (one-line) | Est. | Depends on |
|---|---|---|---|---|
| 1 | **Multiplayer Skeleton** | Colyseus rooms, room codes, lobby, 10 tanks, 1 weapon (Baby Missile), wind+gravity, terrain carving, single-round, win detection, decent Cartoon-Illustrative graphics. | 2–3 wk | — |
| 2 | **Damage & Weapon Variety** | Core 5 weapons (Missile, Baby Nuke, Nuke, Funky Bomb, MIRV). HP bars, damage falloff, splash, death explosions, chain kills, weapon-select UI. | 2 wk | Phase 1 |
| 3 | **Economy & Shop** | Cash per round (damage + kills + survival bonus), shop screen between rounds, inventory, multi-round matches (host configurable 1–20), match-end scoreboard. | 2 wk | Phase 2 |
| 4 | **Defenses & Movement** | All 5 shields + Parachute + Battery + Patriot. Tank fuel/driving. Falling-tank damage rule. **Netcode migration to tick-stream.** | 1–2 wk | Phase 3 |
| 5 | **Terrain Variety & Walls** | All 9 terrain types (Mountains, Hills, Valleys, Cliffs, Crater, Sky High, Plateau, Flat, Random). All 4 wall modes (None / Wrap / Reflect / Absorb). | 1 wk | Phase 1 |
| 6 | **Specialty Weapons** | Remaining 25 weapons — Roller, Leapfrog, Napalm, Dirt variants, Sandhog, Tunneler, Plasma, Laser, Fireball, MIRV variants, Death's Head, Pineapple, Triple Warhead, etc. | 2–3 wk | Phase 4, 5 |
| 7 | **AI Opponents** | 5 difficulties (Moron, Shooter, Pyro, Cyborg, Bouncer). Slot-filling in lobby. AI for full weapon set + walls + economy. | 1–2 wk | Phase 6 |
| 8 | **Visual Polish** | Hats library expanded, parallax skies, time-of-day variants, full per-weapon particle systems, tank death animations, screen shake tuning, final tank/turret art. | 2 wk | Phase 7 |
| 9 | **Audio** | SFX library (per-weapon, per-explosion, per-shield-impact, ambient wind), music tracks (lobby / match / victory), voice clips ("MOMMY!"). | 1 wk | Phase 8 |
| 10 | **Mobile & Accessibility** | Touch controls (virtual joystick, swipe weapon select), colorblind palettes (deuteranopia/protanopia/tritanopia), reduced-motion mode, keyboard-only navigation. | 1–2 wk | Phase 8 |
| 11 | **Stability & Launch** | Reconnect with ghost-AI takeover, spectator mode, match replays (binary serialization + playback), load testing (target 100 concurrent matches/instance), deployment (Fly.io or similar), monitoring, error reporting. | 1–2 wk | Phase 10 |

**Total estimate: 16–22 weeks of focused solo-engineer time.**

## Per-phase deliverables (template)

Each phase spec produces:

1. A design doc in `docs/superpowers/specs/YYYY-MM-DD-phase-N-<topic>-design.md`.
2. An implementation plan (from `superpowers:writing-plans`) in `docs/superpowers/plans/`.
3. Code that passes the phase's acceptance criteria.
4. Test coverage meeting the cross-phase targets.
5. A brief "what shipped" note appended to `CHANGELOG.md`.

## Open questions deferred to later phases

- **Hosting / deployment target.** Phase 1 will run on localhost; pick a host in Phase 1 or Phase 11.
- **Match replay format.** Schema versioning. Phase 11.
- **Persistent accounts / cosmetics.** Out of scope for v1; revisit post-launch.
- **Domain / branding.** "Scorched Earth" name use — research trademark status; may need a riff name. Pre-Phase 11.
- **Monetization.** Out of scope for v1. Free, ad-free.
- **Internationalization.** Out of scope for v1.
